# 仓储选址优化战略报告规范

*本 Skill 旨在确保输出的供应链咨询报告结构严谨、逻辑闭环且具备决策价值。*

## 何时使用

当 Agent 完成了对人口、物流、土地等多源数据的分析，准备生成最终交付报告时，**必须**参考此规范。

## 核心要求

1.  **严禁碎片化**：必须输出一份逻辑连贯的完整报告，而非零散的问答。
2.  **决策导向**：避免单纯的数据堆砌，每一组数据分析后必须给出“这对选址意味着什么”的结论。
3.  **格式统一**：严格遵循下述 Markdown 结构。

## 报告生成模板

请根据之前的分析结果，填充以下结构：

### 1. 执行摘要 (Executive Summary)

*   **项目核心结论**：用精炼的语言（100字以内）概括最终推荐的选址结果。
*   **关键决策依据**：列出支持该决策的 3-5 个最核心的数据支撑点（例如：覆盖周边 X% 人口，预计节约 Y% 运输成本）。

### 2. 项目背景与目标 (Context & Objectives)

*   简述商业背景、覆盖区域范围及优化的核心目标（如：时效优先、成本优先或平衡策略）。

### 3. 综合选址分析 (Comprehensive Site Analysis)

*(此部分需展示深度的分析洞察)*

*   **3.1 市场需求版图 (Demand)**：
    *   描述人口密度分布与核心消费热点。
    *   **结论**：明确指出“需求重心”在哪里，以及选址需靠近哪些高购买力区域。
*   **3.2 物流与劳动力资源 (Supply)**：
    *   展示交通便利性评估与关键枢纽位置。
    *   分析劳动力成本与可用性。
    *   **结论**：指出哪里的运营效率最高且人力资源充足。
*   **3.3 土地条件与限制因素 (Feasibility)**：
    *   总结可用地块筛选结果。
    *   **结论**：列出主要的政策限制和自然灾害避让区，说明“哪些地方绝对不能去”。

### 4. 选址优化方案推演 (Optimization Scenarios)

*(提出 2-3 个具体的建议方案，每个方案需包含以下内容)*

*   **方案 A（推荐方案 - 综合最优）**：
    *   **选址位置**：明确的地理区域描述。
    *   **核心优势**：详细说明其在需求覆盖、物流成本和风险控制上的平衡点。
    *   **预期效益**：量化预测（如：预计覆盖周边 X 公里内 Y 万人口）。
    *   **潜在挑战**：诚实地指出该方案可能面临的小问题（如地价稍高）。

*   **方案 B（替代方案 - 如：成本导向）**：
    *   **选址位置**：明确的地理区域描述。
    *   **核心优势**：说明该方案在特定维度（如极低地价或人力成本）上的独特价值。
    *   **妥协点**：说明相比方案 A，该方案牺牲了什么（如配送时效）。

*   **方案 C（可选方案 - 如：时效导向）**：
    *   *(如果有针对特定战略目标的备选方案，请在此描述)*

### 5. 方案对比与最终建议 (Comparison & Final Recommendation)

*   **5.1 决策矩阵 (Decision Matrix)**：
    *(使用 Markdown 表格形式)*
    | 评估维度 | 方案 A | 方案 B | 方案 C (如有) |
    | :--- | :--- | :--- | :--- |
    | 物流成本 | ... | ... | ... |
    | 市场覆盖率 | ... | ... | ... |
    | 劳动力保障 | ... | ... | ... |
    | 建设风险 | ... | ... | ... |

*   **5.2 最终战略建议**：
    *   明确给出唯一的最终选址决定，并重申最具说服力的理由。

*   **5.3 后续行动路线图**：
    *   为客户提出下一步落地的具体建议（如：启动土地尽调、接触当地招商局、进行环境评估等）。
