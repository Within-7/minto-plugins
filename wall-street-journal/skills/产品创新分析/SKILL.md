# 产品创新分析报告格式规范

*本 Skill 旨在确保输出的商业战略报告结构严谨、逻辑闭环且具有高度的行动指导意义。*

## 何时使用

当 Agent 完成了对产品创新的技术、商业及趋势的多维分析，准备生成最终交付报告时，**必须**参考此规范。

## 核心要求

1.  **结构化**：严格遵循以下报告模板，不要遗漏章节。
2.  **洞察导向**：不要简单罗列新闻事实，必须包含“这意味着什么”的深度见解。
3.  **专业语调**：保持客观、专业、数据驱动的叙事风格（类似 WSJ 深度报道或麦肯锡简报）。

## 报告生成模板

请忽略中间思考过程的冗长展示，直接输出一份结构化文本报告。报告必须包含以下章节：

### 1. 执行摘要 (Executive Summary)

*   **事件概览**：用精炼的语言总结创新事件的核心事实。
*   **颠覆性评估**：一针见血地指出该创新对行业的意义。
*   **战略推荐评级**：给出整体建议（如：重点关注 / 快速跟进 / 战术模仿 / 规避风险）。

### 2. 360度创新解构 (The 360-Degree Analysis)

*(在此部分，将分析的基础事实结构化展示)*

*   **2.1 技术面 (Technical)**：
    *   核心技术突破点与护城河评估。
    *   技术成熟度评级（概念/原型/量产）。
*   **2.2 商业面 (Commercial)**：
    *   目标市场定位与用户画像变化。
    *   盈利模式与核心商业策略（进攻/防御）。
*   **2.3 市场面 (Market)**：
    *   主要竞争对手的反应与应对措施。
    *   关键舆情情感倾向与外部风险雷达。

### 3. 核心洞察与逻辑推演 (Key Insights)

*(重点分析各维度如何相互作用，揭示深层逻辑)*

*   分析技术壁垒与商业定价的关联。
*   揭示竞争格局对技术路线选择的倒逼作用。
*   总结表面现象背后的行业博弈逻辑。

### 4. 未来发展方向与战略预判 (Future Outlook)

*   **创新归属**：明确该产品顺应了哪种宏观行业趋势。
*   **演进路线图 (Roadmap)**：
    *   **短期（1年内）**：功能迭代或市场动作预测。
    *   **中期（1-3年）**：商业模式调整或生态扩张预测。
    *   **长期（3年以上）**：潜在的跨界融合或终局形态。
*   **下一代猜想**：描述“下一代”产品可能具备的特征。

### 5. 战略启示 (Strategic Implications)

*(针对行业玩家或潜在进入者，提供具体的行动建议)*

*   **对于竞品**：具体的防御或反击策略（如：差异化切入）。
*   **对于投资者/高管**：评估该项目成功与否的关键监控指标 (KPIs)。
*   **对于合作伙伴**：如何借势该创新进行生态互补。

### 附录：关键数据概览 (可选)

*   (如有) 核心参数对比表或市场预测数据。
