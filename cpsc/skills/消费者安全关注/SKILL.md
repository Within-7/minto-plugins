# 消费者安全分析报告格式规范

*本 Skill 旨在确保输出的消费者安全分析报告结构严谨、数据可视化清晰且具有商业战略价值。*

## 何时使用

当 Agent 完成了对 CPSC 数据的分析、分类和策略制定，准备生成最终交付报告时，**必须**参考此规范。

## 核心要求

1.  **结构化**：严格遵循以下报告模板。
2.  **可视化替代**：由于无法直接绘图，请使用 Markdown 表格和清晰的文本描述来呈现数据分布。
3.  **行动导向**：改进建议必须具体、可执行。

## 报告生成模板

请直接输出一份结构化、专业、严谨的文本报告。报告必须包含以下章节：

### 1. 执行摘要 (Executive Summary)

*   **综述**：简要说明报告目的，概括数据反映的整体安全状况。
*   **关键发现 (Key Findings)**：列出 3 个最核心的安全痛点（高频或高危）。
*   **核心建议 (Top Recommendations)**：提炼 3 个最具战略价值的改进方向。

### 2. 引言 (Introduction)

*   **背景与数据源**：简述分析背景及使用的数据类型（如消费者投诉、召回记录、伤害报告）。

### 3. 消费者安全关注点全景 (Overview of Consumer Safety Concerns)

*请使用表格形式展示识别出的主要关注点：*

| 核心关注点描述 | 涉及部件/场景 | 危害类型 | 严重程度 |
| :--- | :--- | :--- | :--- |
| [示例：电池过热] | [示例：充电/电池仓] | [示例：热风险] | [示例：严重] |
| ... | ... | ... | ... |

### 4. 深度影响分析 (Impact Analysis)

*   **4.1 频率与分布统计**
    *   *在此处描述最常见的安全问题类型占比，例如：“数据显示电气风险占比最高，达到XX%...”*
*   **4.2 风险严重性评估**
    *   重点阐述被标记为“严重”或“关键”的高风险点及其对品牌声誉的潜在威胁。
    *   *展示高风险点列表：*

| 安全关注点 | 提及频率 | 潜在用户影响范围 | 风险等级 |
| :--- | :--- | :--- | :--- |
| ... | ... | ... | ... |

*   **4.3 趋势洞察** (如数据支持)
    *   分析安全问题的演变趋势（如：某类问题在特定月份激增）。

### 5. 产品改进方向建议 (Product Improvement Recommendations)

*   **5.1 总体策略原则**
    *   简述改进的指导思想（如“合规优先，预防为主”）。
*   **5.2 具体落地建议**
    *   *针对主要风险点，提供详细建议表：*

| 针对的安全关注点 | 根本原因推测 | 具体改进措施 | 改进类型 (设计/材料/说明等) | 优先级 |
| :--- | :--- | :--- | :--- | :--- |
| [示例：电池过热] | [示例：散热设计不足] | [示例：增加石墨散热片] | [示例：设计变更] | [示例：高] |
| ... | ... | ... | ... | ... |

*   **5.3 重点建议深挖**
    *   挑选 1-2 个高优先级建议，详细阐述其实施路径和预期商业价值。

### 6. 战略洞察与未来展望 (Strategic Insights)

*   **商业影响**：论述安全问题对出海战略、合规成本及品牌信任度的影响。
*   **行动路线图**：为咨询客户提供下一步行动建议（如“建立专项测试实验室”、“优化供应链质检”）。

### 附录 (Appendix)

*   数据来源说明或补充的技术细节。
