# 竞品召回分析报告格式规范

*本 Skill 旨在确保输出的竞品分析报告结构严谨、重点突出，且具有高度的可读性。*

## 何时使用

当 Agent 完成了对竞品召回数据的分析、影响评估和弱点识别后，准备生成最终交付报告时，**必须**参考此规范。

## 核心要求

1.  **结构化**：严格遵循以下 Markdown 报告模板。
2.  **可视化**：关键数据（如召回列表）必须使用表格展示。
3.  **战略性**：报告的重心应放在“弱点识别”与“战略建议”上，而非单纯的数据罗列。

## 报告结构概览

1.  **报告摘要 (Executive Summary)**
2.  **召回事件概览 (Recall Events Overview)**
3.  **召回事件详细分析 (Detailed Analysis)**
4.  **竞争对手弱点识别与分析 (Competitor Weaknesses Analysis)** - *战略重点*
5.  **结论与战略建议 (Conclusion & Strategy)**

## 报告生成模板

请忽略中间思考过程，直接输出一份结构清晰的 Markdown 文本报告。报告必须包含以下章节：

### 报告标题：[时间范围] [产品类别] 竞争对手 [竞争对手名称] 产品召回分析报告

#### 1. 报告摘要 (Executive Summary)

*   **分析综述**：简要概述本次分析的目的、覆盖的竞争对手及时间范围。
*   **核心发现**：一针见血地指出竞争对手最致命的 1-2 个系统性弱点。
*   **战略结论**：用一句话总结这些弱点对市场格局的潜在改变。

#### 2. 召回事件概览 (Recall Events Overview)

*   **统计数据**：
    *   报告涉及的召回事件总数、涉及产品总数量。
    *   （如有）召回事件的时间分布趋势描述。
*   **关键召回事件列表**：
    *   *请务必使用 Markdown 表格展示最重要的召回事件（按重要性或时间排序）。*

| 产品型号 | 召回日期 | 召回原因简述 | 召回数量 | 监管机构 |
| :--- | :--- | :--- | :--- | :--- |
| [型号A] | YYYY-MM-DD | [简短描述] | [数量] | [如CPSC] |
| [型号B] | ... | ... | ... | ... |

#### 3. 召回事件详细分析 (Detailed Analysis)

*   **原因类型分布**：
    *   分析导致召回的主要原因类别（如：设计缺陷占比XX%，制造问题占比XX%）。
    *   描述哪类问题是最高频的“痛点”。
*   **市场与声誉影响评估**：
    *   总结媒体和公众对这些事件的整体反应。
    *   特别指出那些引发了重大公关危机或消费者信任崩塌的具体事件。

#### 4. 竞争对手弱点识别与分析 (Competitor Weaknesses Analysis)

*(此部分为报告的核心，请详细阐述)*

*   **系统性弱点一：[弱点名称，例如：供应链质量管控失效]**
    *   **弱点描述**：详细解释该弱点的具体表现。
    *   **支持证据**：明确列出支持该判断的召回事件（引用事件的产品型号或日期）。
    *   **潜在长期影响**：分析该弱点如何通过增加成本、降低创新速度或引来监管审查从而损害竞争对手的长期竞争力。

*   **系统性弱点二：[弱点名称，例如：新产品设计验证流程不严]**
    *   **弱点描述**：...
    *   **支持证据**：...
    *   **潜在长期影响**：...

#### 5. 结论与战略建议 (Conclusion & Strategy)

*   **竞争情报总结**：重申竞争对手目前在产品安全与质量上的主要短板。
*   **对自身企业的战略启示**：
    *   **攻**：如何利用对手的弱点抢占市场（例如：针对对手的电池安全问题，强调我方产品的电池认证优势）。
    *   **守**：从对手的错误中吸取教训，我方应加强哪些环节的管控（例如：加强特定原材料的供应商审核）。

#### 附录
*   **完整数据索引**：如有未在正文中展示的次要数据或细节，可在此列出。
