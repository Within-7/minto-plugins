# 全球产品安全趋势综合报告格式规范

*本 Skill 旨在确保输出的合规与战略报告结构严谨、重点突出，适合企业C-Level高管阅读。*

## 何时使用

当 Agent 完成了对法规数据、召回记录和市场准入要求的分析，准备生成最终交付报告时，**必须**参考此规范。

## 核心要求

1.  **结构化**：严格遵循以下 Markdown 报告模板。
2.  **商业化语言**：避免纯法律术语堆砌，使用“风险”、“成本”、“上市周期”、“竞争优势”等商业词汇。
3.  **可视化**：尽可能使用 Markdown 表格对比新旧标准或不同市场的要求。

## 报告结构概览

1.  **执行摘要 (Executive Summary)**
2.  **全球产品安全标准变更监测**
3.  **目标市场准入合规指南**
4.  **商业战略与行动建议**
5.  **结论**

## 报告生成模板

请忽略中间分析过程的草稿，直接输出一份结构化报告。报告必须包含以下章节：

### 1. 执行摘要 (Executive Summary)

*   用精炼的语言（200字以内）概括当前全球产品安全监管最核心的趋势（例如：“监管重心正从物理安全向化学物质管控转移”）。
*   列出企业当前面临的 **Top 3 合规挑战**。
*   给出最关键的一项 **战略建议**。

### 2. 全球产品安全标准变更监测 (Global Standard Landscape)

*   **2.1 关键变更清单**：
    *   *请使用 Markdown 表格列出最具影响力的3-5项标准变更。*
    *   | 标准名称/编号 | 变更类型(新增/修订) | 生效日期 | 核心技术变更点 | 受影响产品线 |
    *   | :--- | :--- | :--- | :--- | :--- |
    *   | (示例) ASTM F963 | 修订 | 2025-01-01 | 声学测试距离调整 | 玩具 |

*   **2.2 深度影响分析**：
    *   针对上述表格中的变更，具体说明对产品设计和制造流程的冲击。
    *   *例如：是否需要更换阻燃材料？是否需要增加新的模具？是否延长了测试周期？*

### 3. 目标市场准入合规指南 (Market Access Roadmap)

*   **3.1 核心准入要求 (以美国CPSC为例)**：
    *   列出进入该市场必须满足的“合规三要素”：
        1.  **认证要求**：(如 GCC/CPC 证书的具体内容)
        2.  **测试标准**：(第三方实验室测试的强制性项目清单)
        3.  **标签与文档**：(如永久性追踪标签的内容规范)

*   **3.2 常见合规陷阱 (Compliance Pitfalls)**：
    *   基于违规或召回数据，指出企业在市场准入中最容易犯的错误。
    *   *格式建议：使用无序列表，重点加粗。*

### 4. 商业战略与行动建议 (Strategic Recommendations)

*(基于洞察提出的可执行建议，这是报告的高价值部分)*

*   **4.1 研发与供应链重塑**：
    *   提出具体的 R&D 建议（如：引入“合规设计”DfC流程）。
    *   供应链管理建议（如：如何筛选符合新标准的原材料供应商）。

*   **4.2 风险防御机制**：
    *   建议企业建立怎样的内部监控体系来应对快速变化的法规。
    *   针对潜在召回风险的应急预案建议（如：批次管理的重要性）。

### 5. 结论 (Conclusion)

*   一句话总结：重申合规作为企业核心竞争力的价值，而非仅仅是成本中心。
