# 数字营销策略分析报告撰写规范

## 适用范围
- 面向 **多篇 Digital Commerce 360（DC360）文章组成的输入集**
- 内容聚焦：**已实施的营销渠道调整、内容合作、MarTech 部署、个性化实践、归因变革、新兴形式试点**
- **排除项**：未落地计划（“考虑投资 AI”）、模糊表述（“加强数字化”）、战略愿景（无行为证据）
- 所有洞察/推理必须标注信息类型 + 来源主体 + 年份（如有）+ Article ID

## 核心原则

### 🔍 信息类型标注（强制）
所有陈述必须使用以下四类标签之一：

- **【事实 - 引自 X, 年份】**  
  → 企业已实施的行为、公告或声明（如“Walmart 提升 TikTok 预算 40%”）
- **【数据 - 机构, 年份】**  
  → 第三方调研、统计或效果指标（如“直播转化率提升 22%”）
- **【观点 - 主体】**  
  → 高管、专家或创作者的主观判断（如“我们认为创作者是未来销售终端”）
- **【推断 - 高置信度】**  
  → 单一逻辑步骤 + 白名单常识（如“从一次性合作转为佣金制 → 关系长期化”）
- **【推断 - 中置信度】**  
  → ≥2 条文中线索 或 1 条线索 + 常识，但存在替代解释

> 🔹 年份若未提供，可省略（如【观点 - Coresight 分析师】）  
> 🔹 若来源主体不明（如“业内人士称”），写“文中未提及”  
> 🔹 默认策略：模糊/不确定 → “输入文章中未提供相关信息”

### 📐 固定输出结构（七章，强制执行）
- 每章无信息时写：“输入文章中未提供相关信息”  
- 第七章不得引入新事实、新 Article ID 或新推断  
- 所有条目以列表呈现，每条一行

### 🧠 推理边界
- **允许高置信度推断**：仅当逻辑链在白名单常识内且无需额外假设  
- **允许中置信度推断**：需明确列出依据文章 ID（如 Article #2 + #4）  
- **禁止弱推理**：如“多篇文章提 Amazon → Amazon 主导定价”  

### 📌 可视化增强（仅当数据结构清晰时使用）**  
- 所有图表必须基于本章已提取的【事实】或【数据】  
- 禁止为生成图表而编造、聚合或估算数值  
- 若信息不足以构成表格，**不得强行插入图表**  
- **图表必须紧接在文字条目之后**，作为辅助呈现，不可替代文字分析

### 7. 跨维度数字营销洞察与战略启示  
**核心问题**：综合前六章证据，哪些信号最具战略意义？对品牌/平台有何可行动启示？  
**分析指引**：  
- 提炼 1–3 条高价值洞察，每条包含：  
  1. **洞察名称**（如“创作者关系正在产品化”）  
  2. **证据链**（引用 ≥2 个前文章节的具体发现）  
  3. **战略启示**（1 句，聚焦“应如何行动”或“需警惕什么”）  
- **所有内容必须已在前六章出现**，不得引入新事实、新 Article ID 或外部观点  

> 示例：  
> - **💡 启示：弃购挽回正从邮件单通道转向 SMS+邮件组合**  
>   - 证据链：Ch1（Ulta 翻倍 SMS 预算用于弃购提醒）+ Ch3（Gymshark 集成 Klaviyo 实现多通道自动化）  
>   - 启示：品牌应将 SMS 纳入核心弃购挽回策略，尤其针对高客单品类  

> - **⚠️ 风险：MarTech 堆栈复杂度上升，但归因能力未同步进化**  
>   - 证据链：Ch3（多家企业部署 CDP 与 AI 工具）+ Ch5（仅少数采用增量测试，多数仍依赖最后点击）  
>   - 启示：技术投入需配套测量能力建设，否则 ROI 难以验证  

- 若无足够交叉信号：“输入文章中未提供可整合的跨维度洞察”

## 标注与禁止行为

### 统一标注格式
- 【事实 - 引自 X, 年份】（Article #[ID]）  
- 【数据 - 机构, 年份】（Article #[ID]）  
- 【观点 - 主体】（Article #[ID]）  
- 【推断 - 高置信度】（Article #[ID]）  
- 【推断 - 中置信度】（Article #[A] + #[B]）

### 严禁行为
- 使用“计划”“探索”“有望”等未落地语言  
- 编造来源主体或技术细节  
- 在第七章引入新 Article ID  
- 将【观点】作为【事实】输出  
- 为生成图表而合并模糊信息
